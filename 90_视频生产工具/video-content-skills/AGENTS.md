# Repository instructions

- This repository contains Codex-native local Skills, not a visual workflow application.
- Keep every Skill independently triggerable through its `SKILL.md` description.
- Prefer small deterministic scripts and durable JSON checkpoints.
- Paid generation must have a dry-run and require explicit execution.
- Never store provider API keys in project files or logs.
- Preserve third-party license and provenance notices when adapting upstream Skills.
- Verify generated files structurally and inspect representative media visually before claiming completion.
