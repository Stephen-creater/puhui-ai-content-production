## 严格通过条件

1. 黄色胶带的承载面必须真实可见；
2. 透明膜必须从胶带边缘连续展开；
3. 不得凭空长出第二条胶带或错贴到物体上；
4. 物体、膜、胶带在 10 秒内不得变形、消失、融化或跳变；
5. 技术可播放只是底线，不代表画面合格。

关键帧必须先写入 `keyframe-review.json`，六项检查全部通过且图片哈希未变化，流水线才允许提交视频任务。未生成视频的“0/3”表示三张计划测试素材均未产出，并不等于消耗了三次视频额度。

视频返回后固定抽取 1、3、5、7、9 秒逐帧检查，并将结论与视频 SHA-256 写入 `video-review.json`。任何一帧存在破膜、漏盖、断胶、双层膜片或结构含糊，整条视频拒绝；技术完成和完整解码不能代替画面验收。

两个及以上案例必须使用独立 `variant_scenes`。每条案例分别填写物体种类、物体形态、环境场景、覆盖状态、镜头景别、镜头运动和人物/施工状态；重复提示词或重复七维组合会在读取 API Key 前被流水线拒绝。
