# 差异化场景单元设计

## 目标

场景单元用于积累可组合的视频素材，不再把“同一个提示词重复生成三次”计作三个不同案例。每条视频必须对应一套独立、可追溯的视觉方案。

## 七维公式

每个案例必须填写以下七项，并形成唯一组合：

1. 物体种类 `object_type`；
2. 物体形态 `object_form`；
3. 环境场景 `environment`；
4. 覆盖状态 `coverage_state`；
5. 镜头景别 `shot_scale`；
6. 镜头运动 `camera_motion`；
7. 人物/施工状态 `person_or_work_state`。

物体种类相同并不代表重复，但至少要通过其余维度形成肉眼可辨的场景差异。仅在提示词末尾添加“variation 2”不构成差异化。

## 清单与硬门

- 两条输出不得共用完全相同的首帧提示词或视频提示词；
- 两条输出不得共用完全相同的七维组合；
- 多输出项目必须使用 `variant_scenes`，每个输出独立声明场景；
- 首帧仍须逐张通过完整膜六项检查并绑定图片哈希；
- 视频仍须检查 1、3、5、7、9 秒节点并绑定视频哈希；
- 任一条件不合格，只淘汰该案例，不用重复提示词补位。

上述前三项由流水线在读取 API Key 前机械拦截。

## 素材与垃圾桶

通过案例保留在对应电视或饮水机场景单元。被用户明确否定、关键帧不合格或视频质检不合格的素材统一登记到 `08_场景单元/99_垃圾桶（可回顾）/`。垃圾桶保留来源、淘汰原因和证据路径，但不复制大视频，也不作为后续合格率分母。

每个产品维护独立 Excel 台账。主表一行对应一条视频，七个维度必须拆成七列，并紧接真实生产使用的首帧提示词（文生图）和视频生成提示词（图生视频）；负面提示词、模型、成本、素材哈希和淘汰原因继续作为生产与复盘字段。当前 SOP 不包含直接文生视频，不得为其编造提示词字段。Excel 只承担筛选与复盘；流水线继续以项目内的 JSON 记录为事实源，防止手工表格与真实文件分叉。

面向业务人员的台账默认中文优先：七维字段值、首帧提示词、视频生成提示词和负面提示词全部使用中文，只有模型 ID、哈希和路径等技术标识保留原文。

仅完成策略设计、尚未调用模型的案例必须标记为“待生成”，图片和视频哈希保持为空；不得计入首帧通过数、视频通过数或已完成素材数。
