# 覆盖完成态素材首轮验证

> 验证日期：2026-07-16（America/New_York）
>
> **2026-07-16 纠正：本轮三条全部不合格。** 原验收标准错把“黄色胶条位于膜顶部”当成了充分条件，忽略了胶带必须先黏在实际承载面上。电视场景中，黄色胶带应贴在电视上方的墙面，透明膜再从墙面固定线向下覆盖电视。本文保留作为错误基线，纠正后的对照见 `../coverage-complete-wall-anchor-v2/evaluation.md`。

## 目标

验证“减少施工动作、只展示产品已经覆盖物体的完成状态”是否能降低 Grok 视频素材的严重物理错误和整段废片率。

本轮只测一个场景单元，同一张关键帧、同一条视频提示词、同一模型和参数连续生成 3 次。三条之间唯一的计划外变量是模型随机性。

## 场景单元

| 维度 | 取值 |
| --- | --- |
| 物体种类 | 壁挂电视 |
| 物体形态 | 55 英寸、现代黑色矩形外观 |
| 环境场景 | 局部装修中的现代客厅 |
| 产品状态 | 一体式透明 PE 防尘膜已经完整覆盖电视，黄色胶条位于膜顶部 |
| 人物与动作 | 无人物、无安装、无拉膜、无撕除 |
| 景别 | 中景、视平线 |
| 镜头运动 | 极慢推进 |
| 输出 | 9:16、720×1280、10 秒、24 fps |

完整正向和负向提示词以 [`project.json`](project.json) 为权威来源。
三个变体在约 1 秒、5 秒、9 秒的对照见 [`review-contact-sheet.jpg`](review-contact-sheet.jpg)，每一行对应一个变体。

## 视觉验收

纠正后的严格产品边界：黄色美纹胶必须与 PE 膜一体连接，并黏在电视上方的墙面承载产品；膜从胶带下缘向下延伸覆盖电视。胶带不得贴在电视边框上，也不得凭空增加侧边或底部胶带。

| 变体 | 严重物理崩坏 | 产品结构 | 时间连续性 | 严格结论 |
| --- | --- | --- | --- | --- |
| `variant-01` | 无 | 左侧凭空出现竖向黄色胶条 | 稳定 | 判废 |
| `variant-02` | 无 | 左侧凭空出现竖向黄色胶条 | 稳定 | 判废 |
| `variant-03` | 无 | 胶条仍贴在电视顶部边框，未由墙面承载 | 稳定 | 判废 |

共同优点：

- 没有手部、拉膜、撕膜和安装顺序错误；
- 电视没有明显变形、移动或穿模；
- 透明膜没有消失、增殖、融化或剧烈逆重力摆动；
- 三条都保持单镜头和缓慢推进，没有跳切。

共同风险：透明膜偏灰、褶皱较密，画面更像“施工保护状态”，不适合承担产品透明度特写，但可承担装修环境 B-roll。

## 首轮结论

- API 任务成功率：3/3；
- 技术可播放率：3/3；
- 严重物理崩坏率：0/3；
- 严格产品结构通过率：0/3；
- 严格可用素材率：0/3（0%）。

首轮只能说明“完成态画面较少出现严重物理崩坏”，不能证明产品用法正确或素材可用。本轮核心错误是首帧就把黄色胶带贴在电视边框上，后续视频只是稳定地延续了错误。纠正实验必须更换首帧，并在胶带与电视之间保留可见裸墙，让“胶带贴墙”可被直接验收。

本轮不追加生成，避免在没有改提示词前重复消耗相同预算。
