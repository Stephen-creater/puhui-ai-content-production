## 自动验收

- 25/25 视觉镜头存在并通过项目技术检查。
- 25/25 分镜旁白存在。
- 25 个视觉资产 ID 均唯一，没有引用旧视觉库或旧 Demo 片段。
- 旁白为 Alex / ElevenLabs `eleven_v3`，逐镜保持原始语速，未使用 `atempo` 拉长。
- 实测旁白语速范围：161.8–243.2 WPM。
- 每个镜头窗口都长于对应旁白；最小尾部余量为 0.22 秒。
- 每条字幕逐字等于该镜头实际送入 TTS 的 `spoken_text`，并与同一镜头窗口同步。
- 5 条成片均为 H.264、1080×1920、24 fps；音频均为 AAC、48 kHz、双声道。
- 所有成片均满足不少于 20 秒、通常不超过 60 秒的规范。

## 人工画面复核

- 五种策略拥有不同人物、场景、机位与脚本，不是仅替换开头文案。
- 黄色美纹胶与透明 PE 膜保持一体式表达。
- 操作顺序统一为先固定黄色胶边，再向下展开透明膜。
- 字幕与顶部标题位于安全区，抽样画面均可读。
- 3 个本地镜头动作比生成镜头更克制，但无结构变形、倒放、错误揭胶或人物漂移，作为零成本收尾优于继续使用被拒绝的镜头。

## 成本控制规则

后续任何付费批次必须在提交前显示并确认：图片数、视频任务数、生成总秒数、并发数、最大重试数和预计费用。已有总体 API 授权不再等同于无限批量授权。重试必须使用 `--task` 配合 `--force-clips` 或 `--force-keyframes`，禁止连带重做已通过素材。执行器现在还强制要求图片任务、视频任务和付费视频秒数的数值上限；缺少上限或实际计划超过上限时，会在读取 API Key 之前拒绝执行。
