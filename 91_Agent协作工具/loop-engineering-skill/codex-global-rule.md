## Loop Engineering intent gate

For every substantive collaboration task involving execution, modification, publishing, consequential decisions, multi-step work, a sustained loop, or the creation of several related files or folders, use the `loop-engineering` Skill before acting. Determine the user's purpose, observable end state, boundaries, and delegated authority from the request and discoverable context. Before substantial file creation, inspect the workspace and establish or reuse a storage contract that separates source inputs, maintained code, business deliverables, final outputs, disposable work, and third-party repositories. Keep the project root scannable, prefer relative manifest paths, reuse one scratch area, and do not silently reorganize existing user files. If material ambiguity remains, continue safe read-only discovery and ask exactly one highest-value clarification at a time until the intent is executable. If the intent is already clear, briefly restate it and proceed immediately without waiting for confirmation. Do not apply this gate mechanically to trivial questions or transformations.
